## 🎯 نظرة عامة

نظام POS متكامل مصمم خصيصاً لمتاجر الملابس، يوفر حلولاً شاملة لإدارة:
- المنتجات والمخزون
- المبيعات والفواتير
- الموظفين والعمولات
- المصاريف والأرباح
- التقارير والإحصائيات

النظام يعمل بشكل كامل online مع Firebase Firestore كقاعدة بيانات، ويوفر واجهة مستخدم عربية سلسة وسريعة.

## 🏗 البنية التقنية

### المعمارية (Architecture)

النظام مبني على معمارية **3-Layer Architecture**:

```
┌─────────────────────────────────────┐
│      UI Layer (Components)          │
│  - Pages, Components, Modals        │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│      Business Logic Layer           │
│  - Hooks, Services, Utils           │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│      Data Access Layer              │
│  - DataLayer, DataReader            │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│      Firebase Firestore             │
└─────────────────────────────────────┘
```

### طبقة الوصول للبيانات (Data Access Layer)

#### DataLayer.js
واجهة موحدة لجميع عمليات الكتابة:
- `add(collectionName, data)` - إضافة مستند جديد
- `update(collectionName, docId, data)` - تحديث مستند
- `delete(collectionName, docId)` - حذف مستند

#### DataReader.js
واجهة موحدة لجميع عمليات القراءة:
- `get(query)` - جلب مستندات باستخدام query
- `getById(collectionName, docId)` - جلب مستند واحد بالـ ID
- `onSnapshot(query, callback)` - الاستماع للتحديثات في الوقت الفعلي

### Collections في Firebase

- `lacosteProducts` - المنتجات
- `dailySales` - المبيعات اليومية
- `reports` - التقارير (الفواتير المقفلة)
- `masrofat` - المصاريف
- `wared` - الوارد (المنتجات الجديدة)
- `employees` - الموظفين
- `users` - المستخدمين
- `debtsPayments` - فواتير البضاعة
- `dailyProfit` - الأرباح اليومية
- `closeDayHistory` - سجل التقفيلات
- `counters` - العدادات (أرقام الفواتير)

## 🚀 التثبيت والتشغيل

### المتطلبات الأساسية
- Node.js 18+ 
- npm أو yarn
- حساب Firebase

### خطوات التثبيت

1. **استنساخ المشروع**
```bash
git clone <repository-url>
cd clothes_shop
```

2. **تثبيت الحزم**
```bash
npm install
# أو
yarn install
```

3. **إعداد Firebase**
   - أنشئ مشروع Firebase جديد
   - فعّل Firestore Database
   - انسخ بيانات الإعدادات من Firebase Console
   - ضعها في `app/firebase.jsx`

4. **تشغيل المشروع**
```bash
npm run dev
# أو
yarn dev
```

5. **فتح المتصفح**
افتح [http://localhost:3000](http://localhost:3000)

### بناء المشروع للإنتاج
```bash
npm run build
npm start
```

## 💡 كيفية عمل النظام

### 1. عملية البيع

```
1. المستخدم يفتح نافذة البيع
2. يبحث عن المنتج بالكود
3. يختار اللون والمقاس (إن وجد)
4. يضيف المنتج للسلة
5. يضيف بيانات العميل (اختياري)
6. يطبق خصم (إن وجد)
7. يحفظ الفاتورة
8. النظام يقوم بـ:
   - إنشاء الفاتورة في dailySales
   - تحديث المخزون تلقائياً
   - حساب الأرباح
   - طباعة الفاتورة
```

### 2. إدارة المخزون

```
عند البيع:
- النظام يقرأ الكمية الحالية
- يخصم الكمية المباعة
- يحدث المنتج في Firebase
- إذا نفدت الكمية، يحذف المنتج تلقائياً
```

### 3. حساب الأرباح

```
الربح = (سعر البيع - سعر الشراء) × الكمية
الربح الصافي = إجمالي الأرباح - المصاريف
```

### 4. تقفيل اليوم

```
عند التقفيل:
1. نقل جميع الفواتير من dailySales إلى reports
2. حفظ dailyProfit
3. حذف مصاريف اليوم
4. إنشاء سجل في closeDayHistory
```
